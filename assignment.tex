\section{Grid configuration}

The full grid is 71 by 71 but we only put points in the center 50 by 50. The reason for the extra border of 10 cells on each side is because when the boundary walk algorithm runs it needs some empty space around the edges otherwise it would hit the grid border. All the grid numbers are constants at the top of \texttt{grid.ts}:

\begin{lstlisting}[style=tsstyle, caption={Grid constants (\texttt{grid.ts})}]
const CANVAS_GRID_SIZE = 70
const FILL_GRID_SIZE = 50
const SPACING = 11
const POINT_RADIUS = 2
const CANVAS_SIZE = (CANVAS_GRID_SIZE + 1) * SPACING  // 781px
const OFFSET = Math.floor(
  (CANVAS_GRID_SIZE - FILL_GRID_SIZE) / 2)            // 10
\end{lstlisting}

\section{Outer boundary shape}

If we filled the 50 by 50 square with points the boundary detection would have nothing interesting to trace. Instead we generate a random outer boundary that looks like an irregular blob. The way it works is we pick 16 angles evenly spaced around the center of the grid and at each angle we assign a random radius, somewhere between 70\% and 100\% of half the grid size which is 25 cells. Because the radii are different at each angle the shape ends up looking organic instead of just a circle.

The boundary check has the same implementation with interpolation as the hole check:

\begin{lstlisting}[style=tsstyle, caption={Boundary check (\texttt{grid.ts})}]
function interpolateRadius(
  angle: number, radii: number[]
): number {
  let a = angle
  if (a < 0) a += Math.PI * 2
  const n = radii.length
  const sector = (Math.PI * 2) / n
  const idx = Math.floor(a / sector)
  const nextIdx = (idx + 1) % n
  const t = (a - idx * sector) / sector
  return radii[idx] * (1 - t) + radii[nextIdx] * t
}

function isInsideBoundary(
  col: number, row: number, radii: number[]
): boolean {
  const dx = col - CENTER
  const dy = row - CENTER
  const dist = Math.sqrt(dx * dx + dy * dy)
  return dist <= interpolateRadius(
    Math.atan2(dy, dx), radii)
}
\end{lstlisting}

This is the same \texttt{interpolateRadius} that holes use, only difference is holes have a different center point and fewer control radii (8 to 12 instead of 16).

\section{Step 1: hole creation}

Each hole gets a random base radius (2 to 5 cells) and 8 to 12 control angles, the per-angle radii are randomly scaled between 0.5x and 1.3x the base so each hole looks different. In the UI the user can click directly on a dot in the canvas and a hole gets created at that position.

\begin{lstlisting}[style=tsstyle, caption={Hole creation (\texttt{grid.ts})}]
function createHole(cx: number, cy: number): Hole {
  const baseRadius = MIN_HOLE_RADIUS
    + Math.random() * MAX_HOLE_RADIUS_EXTRA
  const numPoints = MIN_HOLE_CONTROL_POINTS
    + Math.floor(Math.random() * MAX_HOLE_CONTROL_EXTRA)
  const radii: number[] = []
  for (let j = 0; j < numPoints; j++) {
    radii.push(baseRadius
      * (HOLE_RADII_MIN_SCALE
         + Math.random() * HOLE_RADII_RANDOM_SCALE))
  }
  return { cx, cy, radii, numPoints }
}
\end{lstlisting}

When user is done adding holes we need to figure out which cells are actually visible (meaning inside the boundary and also not holes). So we loop every cell in the 50 by 50 area and check both conditions:

\begin{lstlisting}[style=tsstyle, caption={Visible point collection (\texttt{grid.ts})}]
function getVisiblePoints(
  boundaryRadii: number[], holes: Hole[]
): Point[] {
  const pts: Point[] = []
  for (let row = 0; row < FILL_GRID_SIZE; row++) {
    for (let col = 0; col < FILL_GRID_SIZE; col++) {
      if (!isInsideBoundary(col, row, boundaryRadii))
        continue
      const gCol = col + OFFSET
      const gRow = row + OFFSET
      if (isInHole(gCol, gRow, holes)) continue
      pts.push([gCol, gRow])
    }
  }
  return pts
}
\end{lstlisting}

\section{Step 2: outer boundary detection}

We want to trace the outer edge of the point cloud as an ordered list of cells, start by picking the cell that is highest up on the grid and furthest to the left if there are multiple at the same height.

From there we do the walk at the current cell we scan outward in a square ring of distance 1 (the 8 cells immediately around it). For each cell in that ring we evaluate two conditions: is it occupied? And does it have at least one empty neighbor in the up/down/left/right directions? If both are true then this cell is a boundary candidate. We measure the angle we would need to turn clockwise from where we just came from in order to face this candidate, and after checking every cell in the ring we sort candidates by that turn angle, the one with the smallest turn is the most ``clockwise'' neighbor and that is where we evaluate next.

Sometimes ring 1 has no new candidates because every boundary cell at distance 1 was already visited. When that happens we expand to ring 2, which is the square of cells at 2 cells away from current position. We keep expanding up to ring $k$. $k$ controls how far we are willing to jump. Ring 1 always gets priority because we want the boundary to be as tight as possible, larger rings are only a fallback for when there are gaps.

\texttt{findTopLeft} in \texttt{geometry.ts} picks the starting cell:

\begin{lstlisting}[style=tsstyle, caption={Finding the start cell (\texttt{geometry.ts})}]
function findTopLeft(
  points: Point[]
): { point: Point; index: number } {
  let bestIndex = 0
  for (let i = 1; i < points.length; i++) {
    const [x, y] = points[i]
    const [bestX, bestY] = points[bestIndex]
    if (y < bestY || (y === bestY && x < bestX)) {
      bestIndex = i
    }
  }
  return { point: points[bestIndex], index: bestIndex }
}
\end{lstlisting}

The turn angle can be negative or bigger than $2\pi$ depending on the direction so we need to normalize it into $[0, 2\pi)$ with modulo.

\begin{lstlisting}[style=tsstyle, caption={Angle normalization (\texttt{geometry.ts})}]
function normalizeAngle(angle: number): number {
  const TWO_PI = Math.PI * 2
  return ((angle % TWO_PI) + TWO_PI) % TWO_PI
}
\end{lstlisting}

The loop runs until either the walk comes back to the start cell which means we successfully traced the whole boundary, or it cannot find any next cell to go to which means something went wrong and we have to fall back to concave hull:

\begin{lstlisting}[style=tsstyle, caption={Boundary walk (\texttt{grid.ts})}]
function detectOuterBoundary(
  points: Point[], k: number
): Point[] {
  if (points.length < 3) return points
  const occupied = new Set(
    points.map(p => gridKey(p[0], p[1])))
  const { point: start } = findTopLeft(points)
  const boundary: Point[] = [start]
  const visited = new Set(
    [gridKey(start[0], start[1])])
  let currentX = start[0], currentY = start[1]
  let backAngle = Math.PI // pretend we came from west

  for (let iter = 0;
       iter < points.length * 2; iter++) {
    let nextPoint: Point | null = null

    for (let ring = 1;
         ring <= k && !nextPoint; ring++) {
      const candidates: {
        x: number; y: number; turn: number
      }[] = []
      for (let dx = -ring; dx <= ring; dx++) {
        for (let dy = -ring; dy <= ring; dy++) {
          if (Math.max(Math.abs(dx), Math.abs(dy))
              !== ring) continue
          const nx = currentX + dx
          const ny = currentY + dy
          if (!occupied.has(gridKey(nx, ny))) continue
          if (!hasEmpty4Neighbor(nx, ny, occupied))
            continue
          const angle = Math.atan2(dy, dx)
          const turn =
            normalizeAngle(angle - backAngle)
          candidates.push({ x: nx, y: ny, turn })
        }
      }
      candidates.sort((a, b) => a.turn - b.turn)
      for (const c of candidates) {
        if (c.x === start[0] && c.y === start[1]
            && boundary.length >= 3)
          return boundary
        if (!visited.has(gridKey(c.x, c.y))) {
          nextPoint = [c.x, c.y]
          break
        }
      }
    }
    if (!nextPoint) break
    backAngle = Math.atan2(
      currentY - nextPoint[1],
      currentX - nextPoint[0])
    currentX = nextPoint[0]
    currentY = nextPoint[1]
    boundary.push(nextPoint)
    visited.add(gridKey(currentX, currentY))
  }

  if (boundary.length < 3)
    return concaveHull(points, k)
  return boundary
}
\end{lstlisting}

\section{Step 3: hole boundary detection}

To detect the holes we need to separate empty cells that are outside the boundary from empty cells that are inside a hole. We do this with a flood-fill that starts from every empty cells on the four edges of the grid. The fill expands inward through 4-connected empty neighbors (up, down, left, right). Any empty cells that the fill reaches is marked as exterior and any empty cells that the fill does not reach is classified as a hole.

\begin{lstlisting}[style=tsstyle, caption={Exterior flood-fill (\texttt{grid.ts})}]
function findExteriorEmptyCells(
  occupied: Set<string>
): Set<string> {
  const exterior = new Set<string>()
  const stack: Point[] = []
  const minB = OFFSET
  const maxB = OFFSET + FILL_GRID_SIZE - 1

  // seed: empty cells on the 4 grid edges
  for (let x = minB; x <= maxB; x++) {
    for (const y of [minB, maxB]) {
      const key = gridKey(x, y)
      if (!occupied.has(key)
          && !exterior.has(key)) {
        exterior.add(key)
        stack.push([x, y])
      }
    }
  }
  for (let y = minB + 1; y < maxB; y++) {
    for (const x of [minB, maxB]) {
      const key = gridKey(x, y)
      if (!occupied.has(key)
          && !exterior.has(key)) {
        exterior.add(key)
        stack.push([x, y])
      }
    }
  }

  // expand inward
  while (stack.length > 0) {
    const [cx, cy] = stack.pop()!
    for (const [dx, dy] of NEIGHBORS_4) {
      const nx = cx + dx, ny = cy + dy
      if (nx < minB || nx > maxB
          || ny < minB || ny > maxB) continue
      const key = gridKey(nx, ny)
      if (!occupied.has(key)
          && !exterior.has(key)) {
        exterior.add(key)
        stack.push([nx, ny])
      }
    }
  }
  return exterior
}
\end{lstlisting}

With the exterior set computed, we scan every occupied cell and check its 4 neighbors, if any neighbor is empty and not in the exterior set then that occupied cell is on the edge of a hole. This produces a flat list of hole boundary cells. To separate them into per-hole groups we run BFS with 8-connectivity (including diagonals) which gives one cluster per hole.

For each cluster we run a second flood-fill starting from empty cells adjacent to the cluster boundary and expanding through non-exterior empties. This collects all interior cells of that hole and a concave hull is computed around each cluster for visualization.

\texttt{detectHoleBoundaries} is where all of this happens:

\begin{lstlisting}[style=tsstyle, caption={Hole detection pipeline (\texttt{grid.ts})}]
function detectHoleBoundaries(
  visiblePoints: Point[], k: number
): { hulls: Point[][]; emptyRegions: Point[][] } {
  const occupied = new Set(
    visiblePoints.map(p => gridKey(p[0], p[1])))
  const exteriorEmpty =
    findExteriorEmptyCells(occupied)
  const holeBoundary = findHoleBoundaryPoints(
    visiblePoints, occupied, exteriorEmpty)
  const clusters = clusterByConnectivity(
    holeBoundary.points, holeBoundary.keys)

  const hulls: Point[][] = []
  const emptyRegions: Point[][] = []

  for (const cluster of clusters) {
    const emptyPoints = floodFillEmptyRegion(
      cluster, occupied, exteriorEmpty)
    const hullInput = emptyPoints.length >= 3
      ? emptyPoints : cluster
    const hull = concaveHull(hullInput, k)
    if (hull.length >= 3) {
      hulls.push(hull)
      emptyRegions.push(emptyPoints)
    }
  }
  return { hulls, emptyRegions }
}
\end{lstlisting}

The three helpers inside are: \texttt{findHoleBoundaryPoints} which does the neighbor checking to find cells on hole edges. \texttt{clusterByConnectivity} which is the 8-connected BFS for grouping (we specifically use 8 not 4 here because diagonal boundary cells should stay in the same group). And \texttt{floodFillEmptyRegion} which fills inward from one cluster to get its interior cells, these are the cells that step 4 will later turn into new points.

\section{Concave hull (Moreira \& Santos 2007)}

Concave hulls show up twice in our pipeline. In step 2 when the boundary walk cannot close the loop we fall back to concave hull. In step 3 we need a polygon around each hole cluster for drawing. The implementation is similar to the boundary walk except the algorithm works on scattered points instead of grid cells.

The algorithm precomputes a k-nearest-neighbor table, for every point it sorts all other points by squared distance. After that the algorithm picks the bottommost-leftmost point as starting position, the walk goes clockwise: at each step look at $k$ nearest neighbors, compute clockwise turn for each, try them from smallest turn to largest. Before moving to a candidate, verify that the edge from current to candidate does not intersect with any edge already placed in the hull.

\begin{lstlisting}[style=tsstyle, caption={One concave hull attempt (\texttt{geometry.ts})}]
function tryHull(
  points: Point[], knnIndex: number[][],
  startIdx: number, k: number
): Point[] | null {
  const hull: Point[] = [points[startIdx]]
  const used = new Set<number>([startIdx])
  let currentIdx = startIdx
  let prevAngle = Math.PI

  for (let step = 0;
       step < points.length; step++) {
    const neighbors =
      knnIndex[currentIdx].slice(0, k)
    const candidates = neighbors
      .map(ni => {
        const angle =
          ptAngle(points[currentIdx], points[ni])
        const turn =
          normalizeAngle(prevAngle - angle)
        return { idx: ni, angle, turn }
      })
      .sort((a, b) => a.turn - b.turn)

    let found = false
    for (const cand of candidates) {
      if (cand.idx === startIdx
          && hull.length >= 3) {
        if (canCloseLoop(
          hull, points[currentIdx],
          points[startIdx]))
          return hull
        continue
      }
      if (used.has(cand.idx)) continue
      const nextPoint = points[cand.idx]
      if (canExtendTo(
        hull, points[currentIdx], nextPoint)) {
        hull.push(nextPoint)
        used.add(cand.idx)
        prevAngle = ptAngle(
          nextPoint, points[currentIdx])
        currentIdx = cand.idx
        found = true
        break
      }
    }
    if (!found) return null
  }
  return null
}
\end{lstlisting}

\texttt{canCloseLoop} and \texttt{canExtendTo} check whether a new edge would cross any existing hull edge. They use the cross product sign test on the two segments. After \texttt{tryHull} produces a result, \texttt{validateHull} confirms that no edges self-intersect and that all input points lie inside the polygon or within 0.5 pixels of an edge (a tolerance needed for floating point precision).

\texttt{tryHull} can return null if it gets stuck so we try $k$ values from $\max(k_\text{start}, 4)$ up to $\min(n-1, 8)$ and return the first valid hull. If none work we fall back to convex hull from \texttt{d3-polygon}.

\section{Step 4: fill hole points}

We take the empty cell positions that \texttt{floodFillEmptyRegion} found for each hole and add them as new points and checking against a set of existing points so we do not add duplicates:

\begin{lstlisting}[style=tsstyle, caption={Filling holes (\texttt{grid.ts})}]
function fillHolePoints(
  emptyRegions: Point[][],
  existingPoints: Point[]
): Point[] {
  const existing = new Set(
    existingPoints.map(p => gridKey(p[0], p[1])))
  const newPoints: Point[] = []

  for (const region of emptyRegions) {
    for (const pt of region) {
      const key = gridKey(pt[0], pt[1])
      if (existing.has(key)) continue
      newPoints.push(pt)
      existing.add(key)
    }
  }
  return newPoints
}
\end{lstlisting}

\section{Step 5: Delaunay triangulation}

We combine the original visible points with the fill points from step 4 and run Delaunay on everything using \texttt{d3-delaunay}. For every triangle that comes out of Delaunay we test 4 sample points: the centroid (average of the 3 vertices) and the midpoint of each edge. If all 4 are inside the outer hull polygon from step 2 then the triangle is good, otherwise we discard it.

\begin{lstlisting}[style=tsstyle, caption={Triangulation with hull filtering (\texttt{grid.ts})}]
function triangulatePoints(
  allPoints: Point[], outerHull: Point[]
): [Point, Point, Point][] {
  if (allPoints.length < 3) return []

  const coords =
    new Float64Array(allPoints.length * 2)
  for (let i = 0; i < allPoints.length; i++) {
    coords[i * 2] = allPoints[i][0]
    coords[i * 2 + 1] = allPoints[i][1]
  }

  const delaunay = new Delaunay(coords)
  const triangles: [Point, Point, Point][] = []

  for (let i = 0;
       i < delaunay.triangles.length; i += 3) {
    const p0 = allPoints[delaunay.triangles[i]]
    const p1 = allPoints[delaunay.triangles[i + 1]]
    const p2 = allPoints[delaunay.triangles[i + 2]]
    if (isTriangleInsideHull(p0, p1, p2, outerHull))
      triangles.push([p0, p1, p2])
  }
  return triangles
}
\end{lstlisting}
